Analyze the staged git diff and write exactly one Conventional Commit message.

Rules:
- Type must be in English: feat, fix, refactor, docs, style, test, chore, build, ci, perf
- Description must be in Korean
- Keep it concise
- Output only one line
- No quotes, no code fences, no explanation

Format:
<type>: <Korean message>

Examples:
feat: 새로운 블로그 포스트 작성 기능 추가
fix: 모바일 화면에서 레이아웃 깨짐 수정
docs: README.md 프로젝트 설명 업데이트
refactor: 포스트 목록 조회 로직 가독성 개선
chore: package.json 의존성 라이브러리 업데이트
