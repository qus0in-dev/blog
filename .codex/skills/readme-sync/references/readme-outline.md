# README Outline

Use this outline when the repository needs a concise but complete README refresh.

## Recommended Sections

1. What the project is
2. Local development commands
3. Build and deploy flow
4. Content location and authoring rules
5. URL or slug rules
6. Repository-specific maintenance notes

## Repository-Specific Details To Surface

- build-time keyword generation
- OG generation during build
- Astro plus Cloudflare deployment shape
- content publishing scripts
- local skill conventions when they affect maintenance

## Keep Out

- generic framework marketing copy
- stale setup instructions
- duplicate command explanations with no repo-specific value
