# Style Patterns

Use these patterns when prose sounds translated or overly explained.

## Prefer

- direct verbs over abstract commentary
- shorter transitions over explanatory filler
- concrete nouns in headings
- one organizing lens repeated through the article
- scoped claims that match the actual evidence in the draft

## Avoid

- `핵심은`, `본질은`, `중요한 점은` when the sentence can just say the point
- repeated `-것이다`, `-수 있다`, `-에 가깝다`
- vague headings such as `정리`, `핵심`, `이유`
- dramatic setup before the real technical point
- overconfident claims that are only backed by a small personal example

## Rewrite Pattern Examples

- `A에 가깝다` -> `A라고 보면 된다`
- `이 차이는 ... 비롯된 것이다` -> `이 변화 뒤에는 ...가 있다`
- `중요한 점은` -> remove it and start with the claim
