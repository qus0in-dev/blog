# Bump Examples

Use these examples when the correct version level feels ambiguous.

## Patch

- page rendering or layout changed
- metadata or routing behavior changed
- build or publish scripts changed
- runtime config changed
- normal feature or fix shipped without breaking compatibility

## Minor

- a clearly new site capability or major section was added
- a broad new user-facing surface shipped
- the project meaningfully expanded beyond a small incremental change

## Major

- an intentional breaking change affects public behavior or integration points

## Usually No Bump

- `README.md` only
- `src/content/**` only
- `.codex/skills/**` only
- purely editorial docs with no shipped code effect
